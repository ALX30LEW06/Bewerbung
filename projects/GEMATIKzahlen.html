E-Rezept Kennzahlen

Schlanke Swing-GUI, die Kennzahlen der Gematik-API live lädt, filtert und als CSV exportiert – komplett offline lauffähig dank gebündeltem Java-Runtime.

HTTP/JSON · org.json

jpackage · jlink

Projektüberblick

Eine eigenständige Desktop-App, die 15 E-Rezept-Kennzahlen aus der Gematik-API live abruft, mit Datum filtert und als CSV exportiert. Das Runtime ist eingebettet – Empfänger brauchen keine Java-Installation.

Kernfunktionen

  • Live-Abruf von 15 Kennzahlen von Gematik TI-Lage API
  • Asynchrones Laden aktueller Werte pro Kennzahl (HttpURLConnection)
  • Checkbox-Auswahl mit Tooltips, erste 4 Kennzahlen vorgewählt
  • Datumsfilter (Start/Ende) mit Date-Picker & JTable-Anzeige
  • CSV-Export mit Semikolon-Delimiter & deutschem Datumsformat
  • Self-Contained: Gebündeltes JRE (jlink) – keine Installation nötig
  • Dual-Mode: CLI (App.java) + GUI (GuiApp.java)

Technologie-Stack

Java 23 (java.base, java.desktop, java.net.http) Swing GUI (JFrame, JTable, GridBagLayout) org.json (JSONArray, JSONObject) HttpURLConnection (User-Agent, Timeout 10s) CSV Writer (BufferedWriter, DateTimeFormatter) jpackage · jlink · run.bat Launcher

Code-Architektur & Module

Core-Klassen

  • App.java: CLI-Anwendung (CSV-Download)
  • GuiApp.java: Swing GUI mit Checkboxen & Tabelle
  • Config.java: Zentrale Konfiguration
  • HttpUtils.java: HTTP GET mit Error Handling
  • JsonParser.java: Multi-Format Parser
  • CsvWriter.java: CSV Export mit Escaping
  • DateUtils.java: Datum Parsing & Format

API-Integration

  • Endpoint: Gematik TI-Lage API
  • Metrik-IDs: 1000-1026 (15 Kennzahlen)
  • Format A: JSON Array
  • Format B: Grafana Frames (Fallback)
  • User-Agent: Mozilla/5.0 (Java)
  • Cutoff: Daten ab 22.02.2022

GUI-Details & Workflow

Swing-Komponenten

  • GridBagLayout: Zweispalten-Ansicht
  • JLabel[]: Zeigt Werte oder "Lädt..."
  • JButton: DatePicker für Datum
  • JScrollPane: Scrollbar bei vielen Metriken
  • JTable: Vorschau der CSV-Daten
  • DefaultTableModel: Dynamische Tabelle mit Datum + gewählte Metriken
  • JScrollPane: Scrollbarer Bereich für alle Kennzahlen
  • Tooltips: Vollständige Kennzahl-Namen bei Hover (> 34 Zeichen gekürzt)

    • User-Workflow

    • 1. Start: App lädt automatisch aktuelle Werte aller 15 Metriken
    • 2. Auswahl: Checkboxen wählen (Standard: erste 4 = E-Rezepte, Med. Einrichtungen, Apotheken, App-Downloads)
    • 3. Datumsfilter: Start-/Enddatum setzen (DatePicker)
    • 4. Laden: Button klicken → API-Calls für gewählte Metriken
    • 5. Tabelle: Gefilterte Daten nach Datum sortiert in JTable
    • 6. Export: CSV-Button → historical_data.csv mit Semikolon-Delimiter

    Packaging & Auslieferung

    Self-Contained Build

    • JAR mit eingebetteter org.json
    • jlink Runtime (java.base, java.desktop, java.logging, java.net.http)
    • Launcher-Skript `run.bat` für Windows
    • Optionaler jpackage-Launcher (`GuiApp.exe`)
    • Verteilung als 7z/zip – nur entpacken und starten

    Start für Endnutzer

    1. Archiv entpacken
    2. `run.bat` doppelklicken (keine Java-Installation nötig)
    3. Checkboxen wählen, Datum setzen, CSV exportieren

    Technische Besonderheiten

    Multi-Format JSON-Parser

    • Unterstützt 2 verschiedene API-Response-Formate (Format A: Array, Format B: Grafana)
    • Automatischer Fallback: Versucht Format A, dann Format B
    • Flexible Feldnamen: "Anzahl", "value", "count" werden erkannt
    • TreeMap für automatische Datums-Sortierung
    • LocalDate-Parsing mit DateUtils für robuste Konvertierung

    Self-Contained Runtime

    • jlink erstellt minimales JRE (nur java.base, java.desktop, java.logging, java.net.http)
    • JAR mit eingebetteter org.json (keine externen Dependencies zur Laufzeit)
    • run.bat Launcher für einfachen Start ohne Java-Installation
    • Optional: jpackage für native GuiApp.exe (Installer-fähig)
    • Verteilung als 7z/zip nur entpacken und run.bat starten

    Entwicklungs-Workflow

    • test_api.ps1: PowerShell-Script zum Testen neuer Metric-IDs (1001-1026)
    • Dual-Mode: CLI (App.java) für Batch-Downloads, GUI (GuiApp.java) für interaktive Nutzung
    • Config-Driven: Alle IDs & Header in Config.java leicht erweiterbar
    • Error Handling: Try-catch mit aussagekräftigen Fehlermeldungen
    • User-Agent Spoofing: "Mozilla/5.0 (Java)" für API-Kompatibilität

    Learnings & Best Practices

    • Stabiles Packaging ohne externe Abhängigkeiten (Self-Contained)
    • Performantes Laden: Asynchrone Werte-Updates in GUI
    • Benutzerführung: Tooltips, Statusanzeigen, Vorauswahl der wichtigsten Metriken
    • Fehlerrobustheit: API-Ausfälle "n/a" statt Crash
    • CSV-Escaping: Semikolon in Feldern wird korrekt escaped
    • Datumsformat: dd.MM.yyyy für deutsche Nutzer (statt ISO)
